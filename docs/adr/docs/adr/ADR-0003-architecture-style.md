# ADR-0003: Архитектурный стиль и структура проектов

Дата: 2026-01-18
Статус: Proposed

## Контекст
Проект дипломный, нужен понятный, объяснимый и поддерживаемый дизайн.
Есть интеграции (Excel, Telegram), но микросервисы избыточны.

## Решение
- Модульный монолит (один деплой).
- Разделение на слои и отдельные проекты:
  - SubFlow.Domain
  - SubFlow.Application
  - SubFlow.Infrastructure
  - SubFlow.Web
  - tests/SubFlow.Tests (позже)

Правило зависимостей:
- Web -> Application
- Infrastructure -> Application + Domain
- Application -> Domain
- Domain не зависит ни от кого.

## Обоснование
- Чёткие границы упрощают документирование и защиту.
- Легко тестировать use-cases без UI/БД.
- Интеграции вынесены в Infrastructure.

## Последствия
- Нужно поддерживать дисциплину зависимостей.
- Модели EF не должны “протекать” в Web напрямую — через DTO/use-cases.
