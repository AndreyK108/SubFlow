# ADR-0002: Выбор БД и подхода к миграциям

Дата: 2026-01-18
Статус: Proposed

## Контекст
Нужно надёжное хранение данных подписок/событий/уведомлений/пользователей.
Важно удобство разработки в Visual Studio и поддержка миграций.

## Решение
- Использовать SQL Server.
- Для dev: SQL Server LocalDB.
- Для деплоя/демо: SQL Server Express (или обычный SQL Server).
- Подход: EF Core Code First + Migrations.

## Обоснование
- Простая интеграция с ASP.NET Core/Identity.
- Миграции дают воспроизводимость схемы БД.

## Последствия
- В репозитории будут храниться миграции EF Core.
- DbContext живёт в проекте Infrastructure.
